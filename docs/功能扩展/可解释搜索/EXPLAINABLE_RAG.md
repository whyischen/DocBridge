# 可解释检索（Explainable RAG）功能文档

## 概述

可解释检索功能让用户清楚地了解为什么某个文档被检索出来，提供透明的匹配逻辑和详细的评分解释。

## Phase 1: MVP 实现 ✅

### 功能特性

1. **匹配度百分比** - 将相似度分数转换为易懂的百分比
2. **关键词匹配统计** - 显示查询中有多少关键词被匹配，以及每个关键词出现的次数
3. **评分详情** - 分解显示语义相似度和关键词匹配的贡献
4. **综合得分公式** - 透明展示最终得分的计算方式（70% 语义 + 30% 关键词）

### 使用方法

### 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--explain/--no-explain` | 启用/禁用可解释模式 | 启用 |
| `--top-k` | 返回结果数量 | 5 |
| `--threshold` | 相似度阈值 | 0.5 |
| `--rerank` | 启用关键词重排序 | 启用 |

### 组合使用

```bash
# 获取更多结果（默认显示详细解释）
cbridge search "embedding model" --top-k 10

# 提高阈值，只看高质量匹配
cbridge search "embedding model" --threshold 0.7

# 简洁模式，不显示解释
cbridge search "embedding model" --no-explain

# 纯语义搜索（禁用重排序）
cbridge search "embedding model" --no-rerank

# 纯语义搜索 + 简洁模式
cbridge search "embedding model" --no-rerank --no-explain
```

## 技术实现

### 评分机制

**综合得分计算公式：**
```
final_score = semantic_score × 0.7 + keyword_match_ratio × 0.3
```

其中：
- `semantic_score`: 向量相似度（来自 ChromaDB）
- `keyword_match_ratio`: 匹配的关键词数 / 总关键词数

### 关键词提取

使用正则表达式 `\w+` 从查询中提取关键词，支持：
- 中文词语
- 英文单词
- 数字

### 匹配统计

对每个结果统计：
- 每个关键词在文档内容中出现的次数
- 每个关键词在文档 URI 中出现的次数
- 总匹配关键词数

## 常见问题

**Q: 为什么默认启用 --explain？**
A: 可解释性是现代 RAG 系统的重要特性，帮助用户理解检索结果。性能开销极小（~10-15ms），用户体验提升显著。如果不需要详细信息，可以使用 `--no-explain`。

**Q: 如何切换回简洁模式？**
A: 使用 `--no-explain` 参数即可：`cbridge search "查询" --no-explain`

**Q: 为什么有些结果没有关键词匹配？**
A: 这是正常的。向量检索基于语义相似度，即使没有完全匹配的关键词，语义相近的文档也会被检索出来。

**Q: 如何理解 70% 语义 + 30% 关键词的权重？**
A: 这个权重平衡了语义理解和精确匹配。语义相似度捕捉深层含义，关键词匹配确保术语准确性。

**Q: 可以调整权重吗？**
A: 当前版本权重固定。如有需求，可以在后续版本中添加可配置选项。

**Q: --explain 会影响检索结果吗？**
A: 不会。`--explain` 只影响显示方式，不改变检索逻辑和结果排序。
